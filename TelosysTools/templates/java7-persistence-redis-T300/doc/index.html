Tutorial for the Redis Telosys bundle

Note

  • The redis compliance level must be 3.5

  • we use Jedis 2.9.0 like redis java client

    • 1. Project Creation

    • You can start by create a new maven project or start on an existing one.

    • The java compliance level must be 1.7

    2. Initialize Telosys Tools

    • Open the properties window of your project
    • In the left entries, select "Telosys Tools" and fill all project informations in "General", "Packages" and "Folders" tab (right side)
    • Click on the "Apply" button (don't forget this step!)
    • Click on "Init Telosys Tools" in the "General" tab => The "Telosys Tools" directory is now added in the project
    • In the "download" tab, download from this Github repository the "redis-templates-TT3100"
    • Click on "Download selected bundle(s)" and verify the checkbox "Install downloaded bundle(s)" is checked.
    • After this, close the project properties window.

    • In the project, you should now have the "redis-templates-TT3100" directory under "TelosysTools/templates"

    5. Generate the Redis Persistence

    • Click on the "code generation" in the DSL interface
    • Select the "redis-templates-TT3100" templates bundle in the dropdown list => all templates are now visible in right list
    • Select all templates and select "copy static resources"
    • Click on the "Generate" button
    • You should have a successfull message

    • After add the maven depedencies (next part) the project source code should compile successfully

    3. Update maven dependencies

    • Under the "TelosysTools" , open the pom_persistence_redis.txt
    • Copy and paste the dependencies to your project pom (if they are not already exist)

    4. Redis Database Configuration

    • Open the redis configuration file in your project : "src/main/ressources/redis.properties
    • Put your redis configuration
